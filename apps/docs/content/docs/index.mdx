Install <code>uniku</code> once, then choose the identifier that fits the boundary in front of you: a database index, a public URL, or an API payload. Every generator has its own entry point, so a UUID, ULID, TypeID, Nanoid, CUID v2, KSUID, ObjectID, XID, or TSID only adds the strategy your application imports.

## The library contract

The library is ESM-only, built on web standards, and tested end to end across Node.js, Bun, Deno, browsers, and Cloudflare Workers. Its supported `1.x` entry points, validation boundaries, and runtime support policy are defined in the [stability contract](https://github.com/jkomyno/uniku/blob/main/docs/STABILITY.md).

<Callout type="info" title="A CLI companion is available">
  Install <code>@uniku/cli</code> when you need an ID in a terminal, migration, or shell pipeline. It shares the library's formats but has its own release lifecycle.
</Callout>
