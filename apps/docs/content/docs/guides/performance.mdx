The numbers below come from the repository's own benchmark and bundle-size tooling, including the rows where an npm alternative wins. Benchmarks run in CI on every push to `main` (GitHub Actions, `ubuntu-latest`), so they reflect ordinary Node.js execution rather than a tuned environment.

## Generation speed

Each row compares uniku's generator against the npm package it targets: `uuid` for UUID v4/v7, `ulid`, `@paralleldrive/cuid2`, `@owpz/ksuid`, `bson` for ObjectID, `nexid` for XID, `tsid-ts`, `nanoid`, and `typeid-js`.

This snapshot comes from the rolling `main` baseline updated on July 14, 2026. Treat the ratios as directional: shared-runner noise and dependency updates can move them. The [live baseline](https://raw.githubusercontent.com/jkomyno/uniku/gh-benchmarks/baseline.json) is the current source of truth.

| Generator | uniku vs npm |
| --- | ---: |
| ULID | 116× faster |
| UUID v7 | 6.4× faster |
| CUID2 | 6.2× faster |
| TypeID | 3.2× faster |
| TSID | 2.2× faster |
| KSUID | 1.6× faster |
| UUID v4 | 1.2× faster |
| ObjectID | 1.2× faster |
| Nanoid | Roughly equal |
| Nanoid (10 chars) | npm is 1.1× faster |
| XID | npm is 1.7× faster |

The [compatibility benchmark](https://github.com/jkomyno/uniku/blob/main/packages/uniku/__tests__/bench/compat.bench.ts) runs each implementation in a fresh process. CI balances both launch orders across four repetitions and reports the per-row median. The rolling action baseline keeps warmed runtime state and one noisy run from deciding the comparison.

## Bundle size

Every entry point is independently importable and tree-shakeable: importing `uniku/ulid` does not pull in the ULID, TypeID, or KSUID code. Minified and gzipped sizes per entry point:

| Import | Minified + gzipped |
| --- | ---: |
| `uniku/ulid` | ~1.9 KB |
| `uniku/typeid` | ~2.7 KB |
| `uniku/uuid/v7` | ~1.6 KB |
| `uniku/tsid` | ~1.6 KB |
| `uniku/ksuid` | ~1.3 KB |
| `uniku/objectid` | ~1.4 KB |
| `uniku/xid` | ~1.9 KB |
| `uniku/nanoid` | ~1.2 KB |
| `uniku/uuid/v4` | ~1.2 KB |
| `uniku/cuid/v2` | ~1015 B |
| `uniku/cuid2` | ~1.0 KB |
| `uniku/errors` | ~184 B |
| `uniku/generators` | ~101 B |

The CUID v2 entry point imports SHA3-512 from `@noble/hashes`, uniku's one runtime dependency; the size above excludes that external dependency's own weight.

## Reproduce these numbers

Run the benchmark and bundle-size tooling from the repository root:

```sh
pnpm bench:summary
pnpm bundle:summary
```

`bench:summary` runs uniku's generators against the equivalent npm package and prints a comparison table. `bundle:summary` builds each entry point and reports its minified, gzipped size.
