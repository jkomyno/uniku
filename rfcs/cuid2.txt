# CUID2 Specification

Collision-resistant Unique IDentifier (CUID), version 2.

Reference: https://github.com/paralleldrive/cuid2

## Abstract

CUID2 is a secure, collision-resistant identifier designed for horizontal scaling
and binary search lookup performance. Unlike its predecessor (CUID), CUID2 hashes
all entropy sources using SHA3-512 to prevent information leakage and enumeration
attacks. Unlike time-ordered IDs (ULID, UUIDv7), CUID2 is intentionally not
sortable by creation time, which is a security feature.

## Key Properties

- **Secure**: SHA3-512 hashing prevents information leakage
- **Collision-resistant**: High entropy from multiple sources
- **URL-safe**: Base36 encoding (a-z, 0-9)
- **Configurable length**: 2-32 characters (default: 24)
- **Valid identifier**: Always starts with a letter (a-z)
- **Non-predictable**: Cannot enumerate or guess valid IDs

## Use Cases

- Security-sensitive identifiers (user IDs, session tokens)
- Systems where enumeration attacks are a concern
- Applications needing valid JavaScript/CSS identifiers
- Distributed systems requiring collision resistance
- Cases where time-ordering would leak information

**Not recommended for:**
- Database indexes requiring time-ordering (use ULID or UUIDv7)
- Cases where creation time should be extractable

## Binary Format

CUID2 does not define a standard binary format. The identifier is designed
as a string-first format. Unlike UUID (16-byte binary) or ULID (16-byte binary),
CUID2 has no canonical binary representation.

**Why no binary representation:**
- SHA3-512 hashing destroys the relationship between input and output
- Variable length (2-32 chars) makes fixed binary format impossible
- Round-tripping would be meaningless
- Omitting `toBytes`/`fromBytes` maintains semantic correctness

If binary storage is required, encode the string as UTF-8 bytes.

## String Encoding

### Character Set

- **Encoding:** Base36 (lowercase letters a-z, digits 0-9)
- **First Character:** Always a lowercase letter (a-z)
- **Pattern:** `/^[a-z][0-9a-z]+$/`
- **Length:** Configurable from 2 to 32 characters (default: 24)

The first character is always a letter to ensure the identifier is valid as:
- JavaScript variable name
- CSS selector
- HTML attribute value

### Format

```
pfh0haxfpzowht3oi213cqos
|------------------------|
       24 characters
      (default length)
```

### Examples

```
pfh0haxfpzowht3oi213cqos    (24 chars, default)
tz4a98xxat                   (10 chars, custom)
nc6bzmkmd014706rfj2k        (20 chars)
ab                           (2 chars, minimum)
```

## API Design

### Type Definitions

```typescript
type Cuid2Options = {
  length?: number       // 2-32 characters (default: 24)
  random?: Uint8Array   // Custom random bytes for deterministic testing
}

type Cuid2 = {
  (options?: Cuid2Options): string
  isValid(id: unknown): id is string
}
```

### Function Signatures

```
cuid2() -> string
```
Generate a 24-character CUID2 using default settings.

```
cuid2(options) -> string
```
Generate a CUID2 with custom options.
- `options.length`: Length of ID (default: 24, range: 2-32)
- `options.random`: Deterministic random bytes for testing

Throws RangeError if length < 2 or > 32.

```
cuid2.isValid(id) -> boolean
```
Validate CUID2 format.
Returns true if: string, length 2-32, starts with a-z, contains only a-z and 0-9.

## Implementation Notes

### Algorithm

1. **Generate Random First Letter**
   - Select random lowercase letter from alphabet (a-z)
   - Ensures valid programming language identifier

2. **Collect Entropy Sources**
   - Timestamp: `Date.now().toString(36)`
   - Counter: Incrementing session counter in base36
   - Salt: Random entropy string of specified length
   - Fingerprint: Host-specific identifier (32 chars)

3. **Combine and Hash**
   ```
   hashInput = timestamp + salt + counter + fingerprint
   hash = SHA3-512(hashInput)
   base36Hash = bigIntToBase36(hash)
   ```

4. **Format Output**
   ```
   output = firstLetter + base36Hash.substring(1, length)
   ```

   Note: The first character of the hash is dropped to avoid histogram bias
   in the distribution of first characters.

### Entropy Sources

| Source | Description | Purpose |
|--------|-------------|---------|
| Timestamp | Current milliseconds since epoch | Time-based entropy |
| Counter | Incrementing value (random start) | Prevents same-millisecond collisions |
| Salt | Random string (length = output length) | Fresh entropy per ID |
| Fingerprint | Environment-specific hash | Distributed system uniqueness |

### Counter

- Initialized randomly between 0 and 476,782,367
- Incremented with each ID generation
- Provides ~22k hosts before 50% initial counter collision
- Wraps on overflow (safe due to other entropy sources)

### Fingerprint

The fingerprint provides uniqueness across distributed systems:

- **Node.js/Browser:** Hash of `Object.keys(globalThis).toString()` + random entropy
- **Fallback:** Pure random entropy if no global object available

The fingerprint always uses CSPRNG regardless of custom `random` option.

## Validation Rules

```typescript
const CUID2_REGEX = /^[a-z][0-9a-z]+$/

function isValid(id: unknown): id is string {
  return (
    typeof id === 'string' &&
    id.length >= 2 &&
    id.length <= 32 &&
    CUID2_REGEX.test(id)
  )
}
```

A valid CUID2 must:
1. Be a string
2. Be between 2 and 32 characters long
3. Start with a lowercase letter (a-z)
4. Contain only lowercase letters and digits (a-z, 0-9)

## Security Considerations

### Information Hiding

All entropy sources are hashed together, making it impossible to recover:
- Creation timestamp
- Counter value
- Host fingerprint
- Generation sequence

This prevents enumeration attacks where attackers could predict or iterate
through valid IDs.

### Collision Resistance

| Length | Approximate Collision Probability (Birthday Paradox) |
|--------|-----------------------------------------------------|
| 24 (default) | 50% at ~4 x 10^18 IDs |
| 32 (maximum) | 50% at ~6.76 x 10^24 IDs |
| 10 | 50% at ~51,386,368 IDs |

### Not K-Sortable (Intentional)

Unlike ULID and UUID v7, CUID2 is intentionally not sortable by creation time.
This is a security feature that prevents:
- Timing attacks
- Sequential enumeration
- Information leakage about creation order

If you need time-ordered IDs, use ULID or UUID v7 instead.

### Computational Cost

CUID2 generation is intentionally slower than alternatives like Nanoid or ULID
due to SHA3-512 hashing. This provides:
- Resistance to brute-force collision attacks
- More thorough entropy mixing
- Security-appropriate performance trade-off

### Additional Security Notes

- Uses `crypto.getRandomValues()` for all randomness (CSPRNG)
- Do not use CUID2s as cryptographic secrets (still guessable in theory)
- Do not assume CUID2s are unguessable for security-critical auth

## Comparison with Other Formats

| Feature          | CUID2     | UUIDv4    | ULID      | Nanoid    |
|------------------|-----------|-----------|-----------|-----------|
| Size (bits)      | Variable  | 128       | 128       | Variable  |
| String length    | 24 (default) | 36     | 26        | 21 (default) |
| Time-sortable    | No        | No        | Yes       | No        |
| URL-safe         | Yes       | No        | Yes       | Yes       |
| Custom length    | Yes       | No        | No        | Yes       |
| Information hiding | Yes     | Partial   | No        | N/A       |
| Hash algorithm   | SHA3-512  | None      | None      | None      |
| RFC Standard     | No        | Yes       | No        | No        |

### Comparison with CUID (v1)

| Feature | CUID (v1) | CUID2 |
|---------|-----------|-------|
| Hash Function | None | SHA3-512 |
| Timestamp Visible | Yes | No |
| Counter Visible | Yes | No |
| Host Info Visible | Yes | No |
| First Character | Always 'c' | Random a-z |
| Predictable | Yes | No |
| Secure | No | Yes |
| Default Length | 25 | 24 |
| Configurable Length | No | Yes |

**CUID v1 is deprecated and should not be used for new projects.**

## Test Vectors

### Example 1: Default Generation
```typescript
const id = cuid2()
// => "pfh0haxfpzowht3oi213cqos" (24 characters, starts with letter)
```

### Example 2: Custom Length
```typescript
const shortId = cuid2({ length: 10 })
// => "tz4a98xxat" (10 characters)

const longId = cuid2({ length: 32 })
// => "abcdefghij0123456789abcdefghijkl" (32 characters)
```

### Example 3: Validation
```typescript
cuid2.isValid("pfh0haxfpzowht3oi213cqos")  // true
cuid2.isValid("ab")                         // true (minimum valid length)
cuid2.isValid("a")                          // false (too short)
cuid2.isValid("1abc")                       // false (starts with digit)
cuid2.isValid("ABCD")                       // false (uppercase not allowed)
cuid2.isValid("ab-cd")                      // false (hyphen not allowed)
```

### Example 4: Invalid Inputs
| Input | Reason Invalid |
|-------|----------------|
| `cuid2({ length: 1 })` | Length < 2 throws RangeError |
| `cuid2({ length: 33 })` | Length > 32 throws RangeError |
| `""` | Empty string fails validation |
| `"1abcdefghij"` | Starts with digit |
| `"UPPERCASE"` | Contains uppercase letters |

### Example 5: Deterministic Testing
```typescript
// For testing with predictable output
const deterministicId = cuid2({ random: new Uint8Array(64).fill(42) })
// Output is deterministic given same random bytes
```

### Example 6: Format Verification
```typescript
const id = cuid2()
// Properties that always hold:
// - typeof id === 'string'
// - id.length === 24 (default)
// - /^[a-z]/.test(id) === true (starts with letter)
// - /^[a-z][0-9a-z]+$/.test(id) === true
```

## Dependencies

The uniku implementation uses:
- `@noble/hashes/sha3` - SHA3-512 (NIST-standard, security-audited)
- Native `BigInt` - For large number to base36 conversion (no external dependency)

Note: The original reference implementation uses `bignumber.js`, but modern
JavaScript engines support native BigInt which is sufficient for this use case.
