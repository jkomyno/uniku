# Nanoid Specification

Tiny, URL-friendly unique string ID generator.

Reference: https://github.com/ai/nanoid

## Abstract

Nanoid is a tiny, secure, URL-friendly unique string ID generator.
It provides shorter IDs than UUID while maintaining similar collision resistance
(126 bits of entropy vs UUID's 122 bits). Unlike time-ordered IDs (ULID, UUIDv7),
Nanoid generates purely random identifiers with no embedded timestamp.

## Key Properties

- **Compact**: 21 characters vs 36 for UUID
- **URL-safe**: Uses A-Za-z0-9_- alphabet by default
- **Secure**: Uses cryptographically strong random APIs
- **Customizable**: Supports custom alphabets and sizes
- **Edge-compatible**: Works with Web Crypto API (Cloudflare Workers)
- **No modulo bias**: Bitmask rejection sampling ensures uniform distribution

## Use Cases

- URL shorteners and slug generation
- Session tokens and invite codes
- Database IDs where time-ordering is not needed
- Any case requiring short, random identifiers
- File names and temporary identifiers

**Warning:** Do not use Nanoid for React component keys if the ID is generated
on each render. Generate IDs once and store them.

## Binary Format

Nanoid does not have a canonical binary format. Unlike UUID (16 bytes) and
ULID (16 bytes), Nanoid is a string-only format.

**Why no binary representation:**
- The rejection sampling algorithm discards information
- Round-tripping (string -> bytes -> string) is impossible
- Variable alphabet makes byte representation meaningless
- Omitting `toBytes`/`fromBytes` maintains semantic correctness

If binary storage is needed, encode the string as UTF-8 bytes.

## String Encoding

### Default Alphabet

URL-safe 64 characters (power of 2 for efficient generation):
```
ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789_-
```

### Default Configuration

- **Size**: 21 characters (max: 2048)
- **Alphabet**: 64 characters (A-Za-z0-9_-)
- **Entropy**: 126 bits (21 * log2(64) = 21 * 6)

### Custom Alphabets

- Minimum: 2 characters
- Maximum: 256 characters (byte value constraint)
- No duplicate characters allowed
- Printable ASCII only (codes 32-126)

## API Design

### Type Definitions

```typescript
type NanoidOptions = {
  random?: Uint8Array  // Custom random bytes for deterministic output
  alphabet?: string    // Custom alphabet (2-256 printable ASCII chars)
  size?: number        // Length of generated ID (default: 21, max: 2048)
}

type Nanoid = {
  (): string
  (size: number): string
  (options: NanoidOptions): string
  isValid(id: unknown): id is string
}

declare const nanoid: Nanoid
declare const URL_ALPHABET: string
```

### Function Signatures

```
nanoid() -> string
```
Generate a 21-character URL-safe ID using the default alphabet.

```
nanoid(size) -> string
```
Generate an ID of specified length using the default alphabet.
Throws RangeError if size < 0 or > 2048.

```
nanoid(options) -> string
```
Generate an ID with custom size and/or alphabet.
- `options.size`: Length of ID (default: 21)
- `options.alphabet`: Custom character set (default: URL_ALPHABET)
- `options.random`: Deterministic random bytes for testing

```
nanoid.isValid(id) -> boolean
```
Validate that a string contains only characters from the default URL-safe
alphabet. **Note:** Does not validate IDs generated with custom alphabets.
Returns true if: string, non-empty, matches /^[A-Za-z0-9_-]+$/

### Exported Constants

```typescript
export const URL_ALPHABET = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789_-'
```

## Implementation Notes

### Algorithm: Bitmask Rejection Sampling

To avoid modulo bias, nanoid uses bitmask rejection sampling:

```typescript
// Calculate mask: smallest power-of-2 minus 1 that covers alphabet size
const mask = (2 << (31 - Math.clz32((alphabet.length - 1) | 1))) - 1

// Calculate step: how many random bytes to request per batch
// The 1.6 multiplier accounts for rejected values
const step = Math.ceil((1.6 * mask * size) / alphabet.length)

function generate(size: number): string {
  let id = ''
  while (id.length < size) {
    const bytes = crypto.getRandomValues(new Uint8Array(step))
    for (let i = 0; i < bytes.length && id.length < size; i++) {
      const index = bytes[i] & mask
      if (index < alphabet.length) {
        id += alphabet[index]
      }
      // Otherwise reject and continue (no modulo bias)
    }
  }
  return id
}
```

### Fast Path Optimization

For power-of-2 alphabet sizes (2, 4, 8, 16, 32, 64, 128, 256), no rejection
is needed. Each random byte maps directly to a character via bitmask.

The default 64-character alphabet uses this fast path.

### Random Byte Pooling

For performance, implementations may pre-allocate a pool of random bytes
and refill it periodically, rather than calling `crypto.getRandomValues()`
for each ID.

## Validation Rules

```typescript
const NANOID_REGEX = /^[A-Za-z0-9_-]+$/

function isValid(id: unknown): id is string {
  return (
    typeof id === 'string' &&
    id.length > 0 &&
    NANOID_REGEX.test(id)
  )
}
```

**Size parameter:**
- Must be a non-negative integer
- `nanoid(0)` returns empty string
- Maximum: 2048 (prevents memory issues)
- Negative, non-integer, or exceeding max throws `RangeError`

**Alphabet parameter:**
- Minimum 2 characters
- Maximum 256 characters (byte value constraint)
- No duplicate characters allowed
- Printable ASCII only (32-126)
- Validated when provided in options

## Design Decisions

**1. No `toBytes`/`fromBytes` helpers**

Unlike UUID/ULID which have canonical binary representations, nanoid is purely
string-based with no fixed binary format. The rejection sampling algorithm
discards information, making round-tripping impossible. Omitting these helpers
maintains semantic correctness over pattern consistency.

**2. No buffer output mode**

Writing ASCII bytes to a buffer would be semantically meaningless
and add complexity without value.

**3. No `create()` factory**

Pre-computing mask/step provides negligible performance gain (two arithmetic ops).
Adding a factory would duplicate generation logic and increase cognitive load.
Users should call `nanoid({ alphabet, size })` directly.

**4. `isValid` validates default alphabet only**

The `isValid(id)` method checks that all characters are from the default URL-safe
alphabet. It does not validate IDs generated with custom alphabets. This limitation
is documented clearly.

**5. Type guard on `isValid`**

Returns `id is string` for better TypeScript DX, allowing type narrowing after
validation checks.

## Security Considerations

1. **Cryptographic randomness**: Uses `globalThis.crypto.getRandomValues()`
2. **No modulo bias**: Bitmask rejection sampling ensures uniform distribution
3. **Web Crypto API**: Compatible with Cloudflare Workers, no Node.js crypto
4. **Size limit**: MAX_SIZE = 2048 prevents memory/DoS issues
5. **Printable ASCII only**: Rejects control characters in custom alphabets
6. **Not time-ordered**: Cannot be used to infer creation time (unlike ULID)
7. **Not suitable as secrets**: While random, Nanoids are not designed as cryptographic tokens

## Comparison with Other Formats

| Feature          | Nanoid    | UUIDv4    | ULID      | CUID2     |
|------------------|-----------|-----------|-----------|-----------|
| Size (bits)      | Variable  | 128       | 128       | Variable  |
| String length    | 21 (default) | 36     | 26        | 24 (default) |
| Time-sortable    | No        | No        | Yes       | No        |
| URL-safe         | Yes       | No        | Yes       | Yes       |
| Custom alphabet  | Yes       | No        | No        | No        |
| Custom size      | Yes       | No        | No        | Yes       |
| Entropy (default)| 126 bits  | 122 bits  | 80 bits   | ~144 bits |
| RFC Standard     | No        | Yes       | No        | No        |

When to choose Nanoid:
- Need short, URL-safe IDs
- Time-ordering not required
- Want customizable alphabet/size
- Simple, fast generation

## Performance

Expected performance for typical implementations:
- ~3-4M ops/sec for default nanoid (64-char alphabet)
- ~2-3M ops/sec for custom non-power-of-2 alphabet

## Test Vectors

### Example 1: Default Generation
```typescript
const id = nanoid()
// => "V1StGXR8_Z5jdHi6B-myT" (21 characters from URL_ALPHABET)
```

### Example 2: Custom Size
```typescript
const shortId = nanoid(10)
// => "IRFa-VaY2b" (10 characters)
```

### Example 3: Deterministic Output (Testing)
```typescript
// With all-zero random bytes, first characters depend on alphabet
const id = nanoid({ random: new Uint8Array(21).fill(0) })
// Each byte & 63 = 0, so all 'A' characters
// => "AAAAAAAAAAAAAAAAAAAAA"
```

### Example 4: Custom Alphabet
```typescript
const hexId = nanoid({ alphabet: '0123456789abcdef', size: 12 })
// => "4f90d13a42bc" (12 hex characters)
```

### Example 5: Invalid Inputs
| Input | Reason Invalid |
|-------|----------------|
| `nanoid(-1)` | Negative size throws RangeError |
| `nanoid(3000)` | Exceeds MAX_SIZE (2048) throws RangeError |
| `nanoid({ alphabet: 'a' })` | Alphabet < 2 chars throws Error |
| `nanoid({ alphabet: 'aa' })` | Duplicate chars throws Error |
| `nanoid({ alphabet: 'a\x00b' })` | Non-printable char throws Error |

### Example 6: Validation
```typescript
nanoid.isValid("V1StGXR8_Z5jdHi6B-myT")  // true (default alphabet)
nanoid.isValid("4f90d13a42bc")            // true (hex chars are in default)
nanoid.isValid("hello world")              // false (space not in alphabet)
nanoid.isValid("")                         // false (empty string)
nanoid.isValid(123)                        // false (not a string)
```
